## 1. ПРЕДВАРИТЕЛЬНОЕ ПЛАНИРОВАНИЕ

### 1.1. Для типовой планировки

**Перед замером пользователь вносит:**

#### A. Последовательность дверей на этаже

**По маркировке:**
```
31л П1.1.Кв1 Ст+10/15 Лв Сн Скц/Мнт
31л П1.1.Кв1 Ст+10/15 Лв Сн Скц/Мнт (санузел)
31л П1.1.Кв2 Ст+10/15 Лв Сн Скц/Мнт
...
```

**По специальному сокращению:**
```
Кв4 - Квартира 4
Кв5 - Квартира 5
...
```

**Важно:** Разные типы дверей могут делать разные подрядчики!

**Принцип:** Типы дверей, материалы, подрядчики и требования к замерам хранятся в **справочниках**, которые заполняет пользователь.

**Структура справочников:**

```
Справочник типов дверей:
  - Тип двери (название)
  - Материал
  - Подрядчик
  - Требуемое количество замеров
  - Дополнительные поля (влажность, фурнитура, огнестойкость...)

Справочник подрядчиков:
  - Название
  - Контактная информация
  - Типы дверей (связь)
  - Требования к замерам

Справочник требований к замерам:
  - Тип двери
  - Количество замеров ширины
  - Количество замеров высоты
  - Количество замеров диагонали
  - Количество замеров толщины
  - Дополнительные замеры
```

**Пример плана замера (данные из справочников):**
```
Этаж 2, Подъезд 1, Секция 1:
  Квартира 1:
    - Дверь 1: 31л П1.1.Кв1 (Тип: [из справочника], Подрядчик: [из справочника])
    - Дверь 2: 31л П1.1.Кв1 (Тип: [из справочника], Подрядчик: [из справочника])
  ...
```

#### B. Особенности дверей

**Информация по местоположению:**
- Этаж: 2
- Подъезд: П1
- Секция: 1
- Квартира: Кв1

**Информация по двери:**
- Открывание: Лв (левое) / Пр (правое)
- Наличники: Сн (с наличником) / Бн (без)
- Стяжка: Ст+10/15 (есть перед/за проёмом)
- Тип проёма: Дп (до плиты) / Дст (до стяжки)

**Информация об окружающей среде:**
- Стены: Скц/Мнт (камень/монолит)
- Штукатурка: Шт+Лв (слева оштукатурено)
- Угловой проём: УпЛв (угловой, стена слева)

#### C. Таблицы с данными

**Таблица дверей:**
| Маркировка | Ширина (мм) | Высота (мм) | Открывание | Тип |
|------------|-------------|-------------|------------|-----|
| 31л П1.1.Кв1 | 1000 | 2100 | Лв | Входная |
| 31л П1.1.Кв1 | 700 | 2000 | Лв | Санузел |
| 31л П1.1.Кв2 | 1000 | 2100 | Пр | Входная |

**Таблица полов:**
| Этаж | Квартира | Тип пола | Высота пола (мм) | Высота стяжки (мм) | Высота чистового (мм) |
|------|----------|----------|------------------|--------------------|-----------------------|
| 2 | Кв1 | Плитка | 150 | 100 | 50 |
| 2 | Кв2 | Ламинат | 120 | 80 | 40 |

### 2.3. Отметки о перемещении в БД

**Структура таблицы:**
```sql
CREATE TABLE Measurements (
    measurement_id TEXT PRIMARY KEY,
    original_order INTEGER,      -- Исходный порядок (при создании)
    display_order INTEGER,       -- Текущий порядок (для отображения)
    moved_from INTEGER,          -- Откуда перемещено (NULL если не перемещалось)
    moved_to INTEGER,            -- Куда перемещено
    moved_at DATETIME,           -- Когда перемещено
    moved_by TEXT,               -- Кто переместил (user_id)
    move_reason TEXT,            -- Причина перемещения

    -- ... остальные поля
);

CREATE TABLE MeasurementMoves (
    move_id TEXT PRIMARY KEY,
    measurement_id TEXT,
    from_order INTEGER,
    to_order INTEGER,
    moved_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    moved_by TEXT,
    reason TEXT,

    FOREIGN KEY(measurement_id) REFERENCES Measurements(measurement_id)
);
```

## 3. МОДУЛЬ АНАЛИЗА ИЗМЕРЕНИЙ (ОНЛАЙН)

### 3.1. Сопоставление измерений

**Автоматический анализ:**
```
1. Чтение последних N замеров
2. Поиск паттернов:
   ├─ 3 замера ширины → группа ширины
   ├─ 3 замера высоты → группа высоты
   └─ Маркировка → определение двери
3. Сравнение с плановыми данными:
   ├─ Ожидалось: 1000 мм
   ├─ Фактически: 999-1015 мм
   └─ Отклонение: -1...+15 мм (допустимо)
4. Проверка на ошибки:
   ├─ Значение < 600 мм → ❌ Ошибка
   ├─ Разброс > 20 мм → ⚠️ Предупреждение
   └─ Неполная группа → ⚠️ Предупреждение
```

### 3.2. Сопоставление с ручными заметками

**Анализ:**
```
1. Чтение notes_raw (ручные заметки)
2. Чтение notes_processed (распознанные)
3. Сопоставление с измерениями:
   ├─ "ЛП слом" → замер 2208.4 мм (высота левая)
   └─ Вывод: дефект на левой петле (высота 2208 мм)
4. Проверка соответствия:
   ├─ Заметка "узкий проём" → ширина 700 мм
   └─ Подтверждение: 700 мм < 800 мм (узкий)
```

### 3.3. Сопоставление с проектными данными

**Анализ:**
```
1. Чтение плановых данных:
   ├─ Ожидаемая ширина: 1000 мм
   ├─ Ожидаемая высота: 2100 мм
   └─ Допуск: ±20 мм
2. Сравнение с фактическими:
   ├─ Фактическая ширина: 999-1015 мм
   ├─ Фактическая высота: 2208-2229 мм
   └─ Отклонения:
       ├─ Ширина: -1...+15 мм (✅ допустимо)
       └─ Высота: +108...+129 мм (❌ превышает допуск)
3. Вывод:
   ⚠️ Высота превышает проектное значение на 108-129 мм
   ✅ Ширина в допуске
```

### 3.4. Защита от ошибочных измерений

**Автоматическая проверка:**
```
1. При добавлении замера:
   ├─ Проверка диапазона:
   │   ├─ Ширина: 600-2000 мм
   │   └─ Высота: 1800-2500 мм
   ├─ Проверка разброса:
   │   ├─ Предыдущие 3 замера: 1000, 1005, 1010 мм
   │   └─ Новый замер: 1500 мм → ❌ Выброс
   └─ Проверка комплектности:
       ├─ Ожидается 6 замеров на дверь
       └─ Сделано 4 → ⚠️ Неполная группа
2. При обнаружении ошибки:
   ├─ Предупреждение: "⚠️ Значение 1500 мм выходит за диапазон"
   ├─ Предложение: "Удалить замер? [Да] [Нет]"
   └─ Логирование: "Замер помечен как подозрительный"
```

### 3.5. Ассистирование пользователю

**Подсказки:**
```
1. После замера ширины:
   "✅ Ширина: 1000 мм. Теперь замерьте высоту (3 замера)"
2. После 3 замеров высоты:
   "✅ Группа полная (6 замеров). Переходите к следующей двери"
3. При отклонении от плана:
   "⚠️ Высота 2208 мм превышает проектную 2100 мм на 108 мм"
4. При пропуске двери:
   "⚠️ Пропущена дверь: Кв3 (ожидалась после Кв2)"
```

**Навигация:**
```
1. После завершения двери:
   "📍 Следующая: Дверь 2/10 (Кв4 санузел)"
2. При завершении этажа:
   "✅ Этаж 2 завершён. Перейти к этажу 3?"
```

## 4. КОЛИЧЕСТВО ЗАМЕРОВ

### 4.1. Принцип определения количества замеров

**Количество замеров определяется на основе:**
- Типа двери (из справочника)
- Требований подрядчика (из справочника)
- Плановых данных (если есть)
- Предыдущих замеров (для проверки отклонений)

**Стандартная структура:**
```
Справочник требований к замерам:
  - Тип двери (ссылка на справочник типов)
  - Количество замеров ширины (обычно 3)
  - Количество замеров высоты (обычно 3)
  - Количество замеров диагонали (опционально)
  - Количество замеров толщины (опционально)
  - Дополнительные замеры (влажность, фурнитура, огнестойкость...)
```

### 4.2. Адаптивное количество

**Алгоритм:**
```kotlin
fun determineRequiredMeasurements(
    doorType: DoorType,          // Из справочника
    contractor: Contractor,      // Из справочника
    planData: PlanData?,         // Из плановых данных
    previousMeasurements: List<Measurement>  // Из БД
): Int {
    // Получаем требования из справочника
    val requirements = lookupRequirements(doorType, contractor)

    // Базовое количество замеров
    var count = requirements.widthCount + requirements.heightCount

    // Дополнительные замеры
    count += requirements.diagonalCount
    count += requirements.thicknessCount
    count += requirements.additionalCount

    // Если есть отклонения от плана → добавляем замеры
    if (planData != null) {
        val deviation = calculateDeviation(previousMeasurements, planData)
        if (deviation > requirements.tolerance) {
            count += requirements.extraMeasurementsForDeviation
        }
    }

    return count
}
```

### 4.3. Дополнительные замеры по требованиям подрядчика

**Принцип:**
- Подрядчик указывает требования в справочнике
- Приложение читает требования и определяет количество замеров
- Пользователь видит подсказки: "Нужно сделать X замеров"

**Пример структуры справочника:**
```json

        "width_measurements": 3,
        "height_measurements": 3,
        "diagonal_measurements": 2,
        "thickness_measurements": 1,
        "additional_fields": ["огнестойкость", "сертификат"]
      }
    }
  ]
}
```

### 5.1. Таблица Measurements

```sql

    measurement_id TEXT PRIMARY KEY,

    value REAL NOT NULL,
    unit TEXT DEFAULT 'mm',
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,

    notes_raw TEXT,           -- Ручной ввод
    notes_processed TEXT,     -- Распознанные
    notes_analysis TEXT,      -- Анализ
    notes_validation TEXT,    -- Валидация

    media_paths TEXT,         -- JSON с путями к медиа
    is_deleted BOOLEAN DEFAULT FALSE,
    is_suspicious BOOLEAN DEFAULT FALSE,

    FOREIGN KEY(project_id) REFERENCES Projects(project_id),
    FOREIGN KEY(floor_id) REFERENCES Floors(floor_id),
    FOREIGN KEY(section_id) REFERENCES Sections(section_id),
    FOREIGN KEY(object_id) REFERENCES Objects(object_id)
);
```

## 7. ДОПОЛНЕНИЕ ЗАМЕРА УЖЕ ИЗМЕРЕННОЙ ДВЕРИ

### 7.1. Сценарии дополнения

**Когда нужно дополнить замер:**

| Сценарий | Описание |
|----------|----------|
| **Неполная группа** | Сделано 4 из 6 замеров → нужно добавить 2 |
| **Подозрительное значение** | Замер 1500 мм (выброс) → нужно перемерить |
| **Добавление диагонали** | После 6 замеров → добавить 2 диагонали |
| **Дополнительные точки** | После 6 замеров → добавить толщину стены |
| **Новые медиа** | После замера → добавить фото/аудио/видео |

### 7.2. Процесс дополнения

**Шаги:**
```
1. Открыть проект
2. Выбрать этаж → секцию → квартиру → дверь
3. Нажать "[Дополнить замер]"
4. Приложение показывает:
   ✅ Сделанные замеры (6)
   ⚠️ Недостающие замеры (0)
   ➕ Дополнительные замеры (опционально)
5. Сделать дополнительные замеры
6. Приложение обновляет запись двери
```

**Интерфейс дополнения:**
```
┌─────────────────────────────────────────────────────────┐
```

**Автоматическое резервирование:**

