## 1. Общие сведения

### 1.1. Назначение документа

Этот документ описывает требования к мобильному приложению для профессиональных замеров в строительстве (двери, окна, отделка).

### 1.2. Термины и определения

| Термин | Определение |
|--------|-------------|
| **Замер** | Одно измерение расстояния (ширина, высота, диагональ) |
| **Точка замера** | Конкретное место замера (верх/низ/середина) |
| **Объект** | Дверь, окно, проём, помещение |
| **Медиа** | Аудио, фото, видео, привязанные к замеру |
| **Транскрипт** | Текстовая расшифровка аудио |

## 2. Проблемы существующих решений

### 2.1. Bosch Measuring Master

| Проблема | Описание | Влияние |
|----------|----------|---------|
| **Пропажа замеров** | Замеры рандомно исчезают из базы | 🔴 Критично |
| **Сбитое время** | В XLS экспорте время = время экспорта, не замера | 🟠 Высокое |
| **PDF экспорт** | Неудобный формат, нельзя искать | 🟡 Среднее |
| **Одна модель** | Только Bosch GLM 50 C | 🟡 Среднее |
| **Заброшен** | Приложение не обновляется | 🟠 Высокое |

### 2.2. Bosch MeasureOn (новое)

| Проблема | Описание | Влияние |
|----------|----------|---------|
| **Нет списка** | Список замеров убрали из UI | 🔴 Критично |
| **Привязка к геометрии** | Нельзя просто список замеров | 🟠 Высокое |
| **Сложный экспорт** | Данные режутся при экспорте | 🟠 Высокое |

### 2.3. Magicplan

| Проблема | Описание | Влияние |
|----------|----------|---------|
| **Платный** | Подписка $10-20/мес | 🟡 Среднее |
| **Упор на план** | Не про замеры, а про чертёж | 🟡 Среднее |
| **Нет raw-лога** | Нельзя выгрузить сырые данные | 🟠 Высокое |

### 2.4. Leica DISTO Plan

| Проблема | Описание | Влияние |
|----------|----------|---------|
| **Дорогая рулетка** | Требуется Leica (от $200) | 🟠 Высокое |
| **Закрытый протокол** | Сложно подключить свою | 🟡 Среднее |

### 2.5. Вывод по аналогам

**Ни одно приложение не решает задачу:**
- ✅ Простой список замеров
- ✅ С подписями/заметками
- ✅ С нормальным экспортом (время замера!)
- ✅ Со стабильным хранением

## 3. Логика обработки измерений

### 3.1. Поток данных

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   Рулетка    │ ──▶ │  Приложение  │ ──▶ │  Хранение    │
│   (BLE)      │     │  (Android)   │     │  (SQLite+JSON)│
└──────────────┘     └──────────────┘     └──────────────┘
                              │
                              ▼
                     ┌──────────────┐
                     │   Экспорт    │
                     │   (XLSX/CSV) │
                     └──────────────┘
```

### 3.2. Структура замера

**Один замер может иметь:**
- Основное значение (мм)
- Несколько точек (верх/низ/середина)
- Голосовую заметку (аудио + транскрипт)
- Фото/видео
- Привязку к объекту (дверь/окно)
- Привязку к PDF (координаты на плане)

**Пример JSON:**
```json
{
  "measurement_id": "001",
  "object_id": "D_кух",
  "object_type": "door",
  "timestamp": "2026-04-02T14:30:00",
  "points": [
    {
      "point_id": "p1",
      "name": "Ширина (верх)",
      "value": 900,
      "unit": "mm",
      "notes_raw": "левая петля сломана",
      "notes_processed": "ЛП сломана",
      "audio": ["audio_001.mp3"],
      "photos": ["photo_001.jpg"]
    },
    {
      "point_id": "p2",
      "name": "Ширина (низ)",
      "value": 905,
      "unit": "mm"
    }
  ],
  "pdf_ref": {
    "file": "floor_plan.pdf",
    "page": 1,
    "rect": {"x": 0.32, "y": 0.48, "w": 0.05, "h": 0.08}
  },
  "export": true
}
```

### 3.3. Привязка заметок

**Варианты привязки:**

| Вариант | Когда | Как |
|---------|-------|-----|
| **К последнему замеру** | Быстрый режим | Авто |
| **К выбранной точке** | Точный режим | Выбор p1/p2/p3 |
| **С регулировкой** | Пост-обработка | Сдвиг после записи |

### 3.4. Обработка аудио

**Поток:**
```
Запись → Очередь → Whisper → Словарь → БД
```

**Словарь сокращений (полный):**

См. [`abbreviations.json`](abbreviations.json) — 500+ сокращений.

**Пример основных:**
```json
{

  "casing": {

    "пр": "Правое",
    "лв": "Левое"
  },
  "plaster": {
    "шт-": "Не оштукатурены",
    "шт+пр": "Справа оштукатурен",
    "шт-пр10": "Справа не штукатурка, выступ 10мм"
  },
  "wall_material": {
    "мнт": "Монолит",
    "скц": "Камень стеновой",
    "крп": "Кирпич",
    "гб": "Газобетон"
  },
  "room_type": {
    "пом": "Помещение",
    "кв": "Квартира",
    "кк": "Коридор кладовых",
    "лх": "Лифт-холл"
  },

    "дп": "До плиты",
    "п1.1": "Подъезд 1, Секция 1",
    "кв1": "Квартира 1"
  }
}
```

### 3.5. Экспорт

**Форматы:**
- XLSX (основной)
- CSV (для импорта в другие системы)
- JSON (для себя/резерв)

**Важно:**
- Время = timestamp замера, не экспорта
- Порядок = хронологический или по объектам
- Фильтр = только отмеченные на экспорт

## 4. Архитектура приложения

### 4.1. Android (Kotlin)

**Слои:**
- UI (Jetpack Compose)
- Domain (бизнес-логика)
- Data (BLE, БД, файлы)

**Модули:**
- `ble` — подключение к рулетке
- `measurement` — управление замерами
- `transcribe` — распознавание речи
- `export` — экспорт в XLSX/CSV
- `pdf` — работа с PDF (Xodo интеграция)

### 4.2. Windows (C#)

**Назначение:**
- Пост-обработка данных
- Массовый экспорт
- Печать отчётов
- Интеграция с CAD (SolidWorks)

**Технологии:**
- WPF или Avalonia UI
- SQLite (та же БД)
- EPPlus (XLSX)

## 6. Интеграции

### 6.1. Bosch GLM 50 C

**Протокол:** BLE (частично разобран)
**Ресурсы:**
- https://github.com/.../Bosch-GLM50C-Rangefinder
- https://github.com/.../BOSCH-GLM-rangefinder

### 6.2. Xodo PDF

**Режим:** Intent (открыть файл)
**Аннотации:** Square (прямоугольники)
**Чтение:** Парсинг PDF annotations

### 6.3. Whisper (оффлайн)

**Варианты:**
- Whisper.cpp (Android port)
- Vosk (легче, но менее точно)
- Сервер (если телефон слабый)

## 7. Этапы разработки

| Этап | Длительность | Результат |
|------|--------------|-----------|
| **1. BLE + замеры** | 2 недели | Приём измерений, сохранение в JSON |
| **2. Голос** | 2 недели | Запись + Whisper + словарь |

**Итого:** 10 недель (~2.5 месяца)
