## Подтверждённые баги

### 1. Время экспорта ≠ время замера

**Доказательство:**
- 46 замеров за 3 минуты (13:50 — 13:53)
- Физически невозможно сделать 46 замеров за 3 минуты
- **Время = время создания XLS файла**, не время замера

### 2. Нет уникальных имён

- Все замеры по умолчанию называются "Измерение"
- Невозможно различить без контекста

### 3. Нет ID устройства

- Нет информации о том, с какой рулетки пришли замеры

### 4. Подзамеры как отдельные строки

- "a", "b", "c" — отдельные строки без группировки
- Нет связи между подзамерами и результатом

## Выводы для нашего приложения

### Что делать иначе

| Проблема Measuring Master | Наше решение |
|--------------------------|-------------|
| Время экспорта | Хранить timestamp замера отдельно |
| Нет ID | UUID для каждого замера |
| Нет имени устройства | Сохранять имя/MAC рулетки |
| Подзамеры в отдельных строках | Группировать в объект |
| Дефолтные имена | Авто-имена: "Замер 1", "Стена A" |
| Нет порядка | Хранить sequence number |

### Модель данных

```csharp
public record Measurement(
    Guid Id,                          // Уникальный ID
    string Name,                      // "Замер 1" или пользовательское
    double Value,                     // Результат
    string Unit,                      // "mm", "mm²", "mm³", "°"
    DateTime Timestamp,               // РЕАЛЬНОЕ время замера
    MeasurementType Type,             // enum: Distance, Area, Volume...
    List<SubMeasurement> SubMeasurements,  // a, b, c
    string DeviceName,                // "GLM 50C"
    string DeviceMac,                 // "20:0b:16:xx:xx:xx"
    int SequenceNumber                // Порядок замера
);

public record SubMeasurement(
    string Label,                     // "a", "b", "c"
    double Value,
    string Unit,
    DateTime Timestamp
);
```
