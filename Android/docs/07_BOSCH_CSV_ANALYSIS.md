### 1.2. Колонки (с заголовками)

| № | Название | Пример | Тип | Проблема |
|---|----------|--------|-----|----------|
| **A** | Название | `Измерение` или `31л П1.22.Кв11...` | Text | ⚠️ Не везде есть маркировка |
| **B** | Значение | `2188,6` | Number | ✅ Нормально |
| **C** | Единица измерения | `mm` | Text | ✅ Нормально |
| **D** | Дата | `05/03/26` | Date | ⚠️ Только дата |
| **E** | Время | `19:22` | Time | ❌ Все одинаковые! |
| **F** | Вид | `Длина` | Text | ✅ Тип замера |

### 1.3. Формат маркировок

**Пример полной маркировки:** `31 П1.1.Кв1 Ст-6.1/15.1 Пр Сн Скц/Мнт Шт-10`

| Часть | Расшифровка | Пример | Значение |
|-------|-------------|--------|----------|
| **31** | Номер двери | 31 | Дверь №31 |
| **П1.1** | Подъезд.Секция | П1.1 | Подъезд 1, Секция 1 |
| **Кв1** | Квартира | Кв1 | Квартира №1 |
| **Ст-6.1/15.1** | Стяжка | Ст-6.1/15.1 | Нет стяжки перед (-6.1), есть за (+15.1) |
| **Пр** | Открывание | Пр/Лв | Правое/Левое |
| **Сн** | Наличник | Сн/Бн | С наличником/Без наличника |
| **Скц/Мнт** | Стены | Скц/Мнт | Слева камень, Справа монолит |
| **Шт-10** | Штукатурка | Шт-10 | Стены не оштукатурены |

**Подтверждённые сокращения (от пользователя):**
- **Сн** = С наличником
- **Дп** = До плиты (структурный проём)
- **Ст-** = Нет стяжки (перед проёмом)
- **Ст+** = Есть стяжка (за проёмом)
- **Пр** = Правое открывание
- **Лв** = Левое открывание
- **Шт-** = Стены не оштукатурены

**Проблема:** Маркировки есть только у ~185 замеров из 1582 (~12%)

**Важно:** Словарь сокращений предоставляется пользователем (не выдумывать!)

## 3. ВЫВОДЫ ДЛЯ MEASURINGASSISTANT

### 3.1. Что делать правильно

**Наш формат CSV:**

```csv
measurement_id,object_id,point_name,value,unit,timestamp,type,notes_raw,notes_processed
m_001,D_кух_01,Ширина (верх),900,mm,2026-04-02T14:30:00,Length,левая петля сломана,ЛП слом
m_001,D_кух_01,Ширина (низ),905,mm,2026-04-02T14:30:05,Length,,
m_002,D_кух_01,Высота (левая),2100,mm,2026-04-02T14:30:10,Length,,
```

**Сравнение форматов:**

| Поле | Bosch MM | MeasuringAssistant |
|------|----------|-------------------|
| **ID замера** | ❌ Нет | ✅ measurement_id |
| **Объект** | ❌ Нет | ✅ object_id (D_кух_01) |
| **Точка замера** | ❌ Нет | ✅ point_name (Ширина верх) |
| **Время** | ❌ Время экспорта | ✅ timestamp (замера) |
| **Заметки** | ❌ Нет | ✅ notes_raw/processed |
| **Уникальность** | ❌ Все "Измерение" | ✅ Уникальные ID |

### 3.2. Импорт из Bosch CSV

**Алгоритм импорта:**

1. **Прочитать файл** (с авто-маркировками)
2. **Перевернуть порядок** (снизу вверх — от старых к новым)
3. **Найти строки с маркировками** (парсинг по формату)
4. **Сгруппировать по 6 замеров на дверь:**
   - Найти строку с маркировкой
   - Взять предыдущие 5 строк
   - Сгруппировать в 1 объект (6 точек замера)
5. **Для каждой группы:**
   - `object_id` = из маркировки (D_31_Кв1)
   - `measurement_id` = новый (m_001, m_002...)
   - 6 точек: p1-p6 (ширина 3 + высота 3)
   - Распарсить маркировку в поля объекта
6. **Сохранить** с пометкой `imported_from: "Bosch MM"`

**Парсинг маркировки (regex):**
```regex
(\d+) л?                    # Номер двери
\s*
(П\d+\.\d+)                 # Подъезд.Секция
\.
(Кв\d+)                     # Квартира
\s*
(Ст-?\d+\.?\d*/\d+\.?\d*)?  # Стяжка (опционально)
\s*
(Пр|Лв)?                    # Открывание (опционально)
\s*
(Сн|Бн)?                    # Наличник (опционально)
\s*
(\w+/\w+)?                  # Стены (опционально)
\s*
(Шт-\d+)?                   # Штукатурка (опционально)
```

**Пример результата:**
```json
{
  "object_id": "D_31_Кв1",
  "door_number": "31",
  "building_section": "П1.1",
  "apartment": "1",
```

## 4. РЕКОМЕНДАЦИИ

### 4.1. Для экспорта MeasuringAssistant

**Формат CSV:**
- Разделитель: запятая (`,`) или точка с запятой (`;`)
- Кодировка: UTF-8 (совместимость с Excel)
- Дата/время: ISO 8601 (`YYYY-MM-DDTHH:MM:SS`)

**Пример:**
```csv
measurement_id,object_id,point_name,value,unit,timestamp,notes_raw,notes_processed
m_20260402_143000_001,D_кух_01,Ширина (верх),900,mm,2026-04-02T14:30:00,левая петля сломана,ЛП слом
```

### 4.2. Для импорта из Bosch

**Алгоритм:**
1. Прочитать CSV
2. Для каждой строки:
   - Создать `measurement_id` (новый)
   - Взять значение из колонки B
   - Взять дату из колонки D
   - Время = `null` (недоступно)
   - Сохранить с пометкой `imported_from: "Bosch MM"`
3. Предупредить пользователя о потере времени
