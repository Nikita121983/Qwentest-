## 1. RFCOMM (Bluetooth Classic) — Measuring Master

### Формат пакетов

**Отправка (host → device):**
```
[start_byte][command][length]([data])[checksum]
```

**Получение (device → host):**
```
[status][length][data...][checksum]
```

### Команды (подтверждённые)

| Команда | Hex | Описание | Источник |
|---------|-----|----------|----------|
| `measure` | `C0 40 00 EE` | Запуск измерения | philipptrenz, EEVBlog |
| `laser_on` | `C0 41 00 96` | Лазер ВКЛ | philipptrenz, EEVBlog |
| `laser_off` | `C0 42 00 1E` | Лазер ВЫКЛ | philipptrenz, EEVBlog |
| `backlight_on` | `C0 47 00 20` | Подсветка ВКЛ | philipptrenz, EEVBlog |
| `backlight_off` | `C0 48 00 62` | Подсветка ВЫКЛ | philipptrenz, EEVBlog |
| `get_serial` | `C0 06 00 4A` | Серийный номер | EEVBlog |
| `get_firmware` | `C0 04 00 BA` | Версия прошивки | EEVBlog |
| `unknown` | `C0 03 00 74` | ? (ответ: `00 01 00 AE`) | EEVBlog |
| **BLE init** | `C0 55 02 01` | Инициализация BLE | ketan репо, HCI лог |
| **BLE init** | `C0 55 02 F1` | Инициализация BLE (альт.) | HCI лог |
| **BLE cmd** | `C0 56 01 00 1E` | Лазер + замер (BLE) | Nordic DevZone |

### Ответы (статусы)

| Байт | Значение |
|------|----------|
| `00` | OK |
| `01` | Таймаут связи |
| `03` | Ошибка контрольной суммы |
| `04` | Неизвестная команда |
| `05` | Недопустимый уровень доступа |
| `08` | Аппаратная ошибка |
| `0A` | Устройство не готово |
| `0D` | Ошибка доступа + аппаратная |

### Пример ответа на измерение

```
Ответ: 00 04 13 0E 00 00 32
  00       = статус OK
  04       = длина данных (4 байта)
  13 0E    = расстояние little-endian = 0x0E13 = 3603
  00 00 32 = checksum/доп.данные
```

**Расчёт:** `0x0E13 * 0.05 = 180 мм` (EEVBlog) ИЛИ `0x0E13 = 3603 мм` (philipptrenz)

| Имя | UUID | Источник |

| **MIRACULIX2_SERVICE** | `0000FDE8-0000-1000-8000-00805F9B34FB` | MeasureOn APK |
| **MIRACULIX2_CHAR** | `02A6C0D2-0451-4000-B000-FB3210111989` | MeasureOn APK |
| **MIRX2_INFO_SERVICE** | `02A6C1A0-0451-4000-B000-FB3210111989` | MeasureOn APK |
| **LINK_ENCRYPTED_TEXT** | `02A6C1A7-0451-4000-B000-FB3210111989` | MeasureOn APK |
| **CCCD_DESCRIPTOR** | `00002902-0000-1000-8000-00805F9B34FB` | MeasureOn APK |

### Формат BLE уведомлений

**Префикс:** `C0 55 10 06`
**Данные:** байты 7-10 = расстояние (float32 little-endian, в метрах)

```
C0 55 10 06 XX XX XX XX YY YY YY YY
  │  │  │  │  └─ байты 7-10: float32 distance (метры)
  │  │  │  └─ байт 6: тип уведомления
  │  │  └─ байт 5: 0x10 = measurement
  │  └─ байт 4: 0x55 = BLE prefix
  └─ байт 3: 0xC0 = start byte
```

**Типы уведомлений (из HCI лога):**
```
C0 55 10 02 → ? (начало измерения)
C0 55 10 06 → distance (результат)
C0 55 10 0A → ?
C0 55 10 0E → ?
C0 55 10 12 → ?
... до C0 55 10 3E
```

### BLE команда (Nordic DevZone)

```
C0 56 01 00 1E
  1-я запись → активация лазера
  2-я запись → запуск измерения
Ответ → float (расстояние)
```

## 6. Особенности

### TrackMyTools (TMT)
- Устройство шлёт BLE рекламу даже в выключенном состоянии
- Отключается в настройках прибора
- Полезно для быстрого подключения

### Два протокола
- **Measuring Master** → RFCOMM (Classic BT)
- **MeasureOn** → BLE (GATT)
- GLM 50C поддерживает **оба**

### Скорость
- ~4 измерения/сек (EEVBlog)
