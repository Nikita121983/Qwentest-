# Автомаркировка — Полная документация

**Файл:** Обработка.xlsx  
**Лист:** Автомаркировка  
**Размер:** 3000 строк × 191 столбец  
**Формул:** ~141 017  

---

## 1. Назначение листа

Автомаркировка — **ядро обработки**. Принимает сырые данные замеров из «Список измерений фильтр», применяет справочники и правила, генерирует финальную маркировку дверей и все сопутствующие параметры.

## 4. Ключевые логические блоки

### 4.1 Парсинг «Помещение» (BJ:BW → CH:CQ)

**Формула (одна для всех CH-CQ):**
```excel
=IFERROR(INDEX($BJ2:$BW2,
   MATCH(TRUE, BYCOL($BJ2:$BW2,
     LAMBDA(col, SUM(--COUNTIF(col, [ИменованныйДиапазон])))),
   0)), "")
```

**Как работает:**
1. BJ:BW содержат разбитую строку (TEXTSPLIT по разделителю)
2. BYCOL проверяет КАЖДУЮ часть на совпадение с именованным диапазоном
3. MATCH находит ПЕРВЫЙ столбец где есть совпадение
4. INDEX возвращает значение

**Именованные диапазоны:**
- `Сокращение` → CI
- `Открывание` → CL
- `Наличник_сокращение` → CM
- `Материал_стен_сокращение` → CN
- `Угловой_проем_наличие` → CO
- `Штукатурка_в_проеме_наличие` → CP
- `Штукатурка_стен_наличие` → CQ
- и т.д.

### 4.2 Маркировка (AT)

```excel
=IFERROR(SUBSTITUTE(
   IF(CI2<>"", CI2,
     INDEX(Сокращение_маркировки,
       MATCH(IF(ISNUMBER(MATCH(...)), ...), Сокращение_маркировки, 0))),
   "Z", "*"), "")
```

**Логика:**
- Если CI (из парсера) не пуст → используем CI
- Иначе → ищем в словаре `Сокращение_маркировки`
- Заменяем "Z" на "*" (форматирование)

### 4.3 Примечания (CU:DL → DM)

Каждый из BJ:BO проверяется на соответствие `Примечания_сокращение`:
```excel
=IF(OR(COUNTIF(BJ2, Примечания_сокращение)), BJ2, "")
```

DM собирает все:
```excel
=IF(AU2<>0, CONCAT(CU2:DL2), IF(AU2<=0, ""))
```

### 4.4 Итоговое название (H)

```excel
=TRIM(IF(AT2<>"", BH2, IF(AI2<>"", AI2, E2)))
```

**Приоритет:**
1. Если есть маркировка (AT) → итоговая строка BH
2. Если есть AI (fallback) → AI
3. Иначе → E (замер словарь)

## 6. Потенциальные проблемы

### 6.1 Цепочка зависимостей

```
CJ → FM → FP:FT / FV:FZ → FQ/FR/FS → AB → AD/AE/AF → AG → AR → AU → AT → BH → H
```

**15 уровней вложенности.** Опечатка в CJ → непредсказуемый результат в H.

### 6.2 Столбцы BJ:BW (19 штук)

Временный буфер для TEXTSPLIT. Если разделитель (BI) меняется — все 19 столбцов пересчитываются. На 3000 строк = **57 000 формул** для парсинга одной строки.

### 6.3 BYCOL + COUNTIF

Каждый столбец CH-CQ выполняет:
- 19 проверок (BJ:BW) × N элементов в именованном диапазоне
- Это O(19 × N) операций на ячейку
- 13 столбцов × 3000 строк = **~700 000 проверок**

### 6.4 OFFSET для «заполнения пропусков»

```excel
FG2: =IF(B2<>"", IF(NOT(LEN(FI2)=0), FI2, OFFSET(FG2,-1,0)), "")
```

Эта формула ссылается **на саму себя** (OFFSET на предыдущую строку). Это:
- Запрещает параллельный пересчёт
- Может дать `#ССЫЛКА!` если строка 1 пустая
- При вставке/удалении строк ломается

## 7. Рекомендации по улучшению

### 7.1 Быстрые победы (без изменения архитектуры)

| Что | Эффект |
|-----|--------|
| Убрать OFFSET в FG/FH — заменить на `SCAN` | Убирает циклическую зависимость |
| Сократить BJ:BW с 19 до 8 (реально используется ~6) | -33K формул |
| Сократить примечания CU:DL с 16 до 6 | -30K формул |
| Удалить пустые FV:FZ (дубликат FP:FZ) | -15K формул |

### 7.2 Среднесрочные

| Что | Эффект |
|-----|--------|
| Заменить BYCOL+COUNTIF на XLOOKUP | В 3-5 раз быстрее |
| Заменить CONCAT-цепочки на spill TEXTSPLIT | Одна формула вместо 19 |
| Сгруппировать параметры стяжки в один spill-блок | Вместо FE:FL — 2 формулы |

### 7.3 Долгосрочные

| Что | Эффект |
|-----|--------|
| Вынести парсинг (BJ:CQ) в spill-формулу с LET | 1 формула вместо 200K |
| Вынести примечания (CU:DL) в spill | 1 формула вместо 50K |
| Создать «тестовый» лист с 5 строками для проверки формул | Предотвращает ошибки на реальных данных |
