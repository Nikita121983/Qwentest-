# Рекомендации по улучшению конвейера обработки замеров

**Контекст:** Каждый замер — новый объект. Файл-источник меняется, логика нарастает.
**Цель:** Сохранить результат в Excel как сейчас, но улучшить управляемость.

---

## Приоритет 1: Немедленно (без изменений в формулах)

### 1.1 Отключить пересчёт перед сохранением

```
Файл → Параметры → Формулы:
☑ Вычисления: Вручную
☐ Пересчитывать книгу перед сохранением  ← СНЯТЬ
☐ Фоновый пересчёт  ← СНЯТЬ
```

**Эффект:** сохранение на 30-60 секунд быстрее.
**Риск:** минимальный — перед сохранением выходного файла жми F9.

### 1.2 Создать шаблон

Скопируй текущую `Обработка.xlsx` в `Обработка_ШАБЛОН.xlsx`:
- Удали все данные (оставь формулы)
- Это база для новых объектов

**Зачем:** не нужно каждый раз копировать и чистить рабочий файл.

### 1.3 Создать тестовый лист

В `Обработка.xlsx`:
1. Вставь новый лист «Тест»
2. Скопируй 5 строк из «Список измерений фильтр»
3. Скопируй 2-3 формулы из «Расчет» и «Автомаркировка»

**Зачем:** прежде чем менять формулу в основном файле — проверь на 5 строках.

## Приоритет 2: Быстрые улучшения в формулах

### 2.1 Заменить INDEX/MATCH на XLOOKUP

**Было:**
```excel
=INDEX(Маркировка!D:D, MATCH($B4, Маркировка!$B:$B, 0))
```

**Стало:**
```excel
=XLOOKUP($B4, Маркировка!$B:$B, Маркировка!D:D, "")
```

**Почему:** XLOOKUP в 2-3 раза быстрее, не требует MATCH, обрабатывает ошибки сам.

**Где менять:** 200 000+ формул. Делай постепенно:
- Сначала в «Расчет» (самый тяжёлый)
- Потом в «Замер»
- Потом в остальных

**Как:** Ctrl+H → Найти: `=INDEX(` → Заменить: `=XLOOKUP(` → потом доделать руками
Или через `formula_search.py` — найди все INDEX/MATCH, замени по шаблону.

### 2.2 Убрать OFFSET в Автомаркировке

**Проблема:** столбцы FG и FH используют OFFSET на предыдущую строку:
```excel
=IF(B2<>"", IF(NOT(LEN(FI2)=0), FI2, OFFSET(FG2,-1,0)), "")
```

Это запрещает параллельный пересчёт и может сломаться.

**Решение:** если данные всегда идут подряд — замени на SCAN:
```excel
=SCAN("", FI2:FI500, LAMBDA(acc, val, IF(val<>"", val, acc)))
```

Или проще — если можно, убери эту логику совсем.

## Приоритет 3: Среднесрочные

### 3.1 Документировать каждый столбец Автомаркировки

Создай лист «Описание» в Обработка.xlsx:

| Столбец | Название | Когда работает | От чего зависит |
|---------|----------|----------------|-----------------|
| AZ | Угловой проем | Когда CO заполнен | AM, CO |
| CF | Замер+ЭтажСокр. | Когда BJ/CH заполнены | BJ, BK, CH, CI |

**Зачем:** когда добавляешь новый объект — сразу видишь какие столбцы нужны.

### 3.2 Создать «карточку объекта»

Лист «Настройки объекта» в Обработка.xlsx:

| Параметр | Значение |
|----------|----------|
| Объект | Юнтолово 8 корпус |
| Профиль | Стандарт |
| Угловые проемы | Нет |
| Штукатурка в проеме | Нет |
| Пороги | Нет |
| Секций | 2 |
| Этажей | 22 |

Формулы ссылаются на этот лист:
```excel
=IF($Настройки$B$4="Нет", "", ...формула для углового проема...)
```

**Эффект:** Excel не пересчитывает формулы для отключённых параметров.

### 3.3 Группировкаspill-блоков

Вместо 19 столбцов BJ:BW — одна spill-формула:

**Было (в каждой строке):**
```
BJ2: =IFERROR(TEXTSPLIT(E2, BI2), "")
BK2: =IFERROR(TEXTSPLIT(E2, BI2), "")  (2-й элемент)
... ×19
```

**Стало (одна формула):**
```
BJ2: =LET(части; TEXTSPLIT(E2, BI2); части)
```
→ spill автоматически заполняет BJ2:BT2

## Приоритет 4: Долгосрочные

### 4.1 Вынести парсинг Автомаркировки в spill-блок

**Текущая проблема:** BYCOL + COUNTIF для каждого столбца CH-CQ = 700 000 проверок.

**Решение:** один spill-блок с LET:

```excel
=LET(
  части; TEXTSPLIT(CJ2; "/");
  секция;   XLOOKUP(TRUE; COUNTIF(Сокращение; части); части; "");
  помещение; XLOOKUP(TRUE; COUNTIF(Помещния_сокращение; части); части; "");
  ...
  HSTACK(секция; помещение; ...)
)
```

Одна формула вместо 13 × 3000 = 39 000.

### 4.2 Разделить файл на модули

**Сейчас:** один файл 10MB+, 357K формул.

**Можно:**
```
Обработка_ЯДРО.xlsx       — Расчет, Замер, Списки (логика)
Обработка_Автомаркировка  — парсинг, маркировка
Обработка_Выходные        — Дефектные, Шахматки, Минимум
```

Связи через внешние ссылки. Открываешь только нужный файл.

### 4.3 Двойной конвейер: Excel + Python

Если в будущем захочешь ускорить:

```
Прибор → edit.xlsx
           │
     ┌─────┴─────┐
     ▼           ▼
  Excel      Python
  (форма)   (обработка)
     │           │
     ▼           ▼
  Результат ← Python результат.xlsx
```

Python делает тяжёлую обработку, Excel — форму и выдачу.
