## 1. Контекст и проблема

### 1.1. Текущий рабочий процесс

```
Bosch GLM 50C (рулетка BLE)
    ↓
Bosch Measuring Master (Android) — список измерений
    ↓
Экспорт в edit.xlsx (Список измерений)
    ↓
Обработка.xlsx (357K формул, 17 листов)
    ↓
Обработка БФ.xlsx (внешние ссылки)
    ↓
Юнтолово 8 корпус...xlsx (финальный, для менеджеров)
```

### 1.2. Почему Measuring Master, а не MeasureOn

| Measuring Master (старое) | MeasureOn (новое) |
|---------------------------|-------------------|
| ✅ Список измерений с подписями | ❌ Упор на геометрию/планы |
| ✅ Экспорт в XLS | ❌ Списка измерений нет |
| ✅ Работает | ❌ Замеряет, но не ведёт лог |

### 1.3. Основные боли

| Боль | Описание |
|------|----------|
| **Медленный пересчёт** | 357K формул → 2-5 минут на обновление |
| **Сложность изменений** | Новая логика = правка в тысячах ячеек |
| **Мёртвые вычисления** | 69% формул считают пустоту |
| **Хрупкость** | Опечатка → сдвиг ссылок → тихая ошибка |
| **Ручной экспорт** | Сохранить как → Разорвать связи → Сохранить |
| **Потеря данных в Measuring Master** | Измерения рандомно пропадают из БД |
| **Время в экспорте** | В XLS время меняется, в PDF — правильное |

## 2. Три режима работы приложения

### 2.1. Массовый замер (основной)

**Для:** типовые объекты (жилые дома, этажи с одинаковой планировкой)

**Поток:**
```
BLE → Поток измерений → ScenarioEngine → HeightAnalyzer 
     → SignatureBuilder → Excel-экспорт
```

**Ключевые особенности:**
- Заранее заданная последовательность дверей (этаж → секция → квартира → дверь)
- Автоматическая привязка замеров к дверям по порядку
- Контроль паттерна (3×3 → 2×2 → 1×1)
- Адаптивное упрощение при стабильных данных

### 2.2. PDF-разметка (нестандартные объекты)

**Для:** школы, сады, подвалы, техэтажи — где нет типовой последовательности

**Поток:**
```
PDF с аннотациями XODO → Тап по зоне → Привязка замера → Excel
```

**Ключевые особенности:**
- PDF-план с цветными прямоугольниками (двери)
- Тап по прямоугольнику → активная зона
- Замеры привязываются к зоне, а не к порядку
- Порядок обхода произвольный
- Подсветка: 🔴 не замерен → 🟡 частично → 🟢 завершён

### 2.3. Разовый замер (быстрый)

**Для:** 1-2 двери без документации, проверка, контрольный замер

**Поток:**
```
Ручной выбор параметров → BLE замер → Краткий отчёт
```

**Ключевые особенности:**
- Быстрый выбор: тип двери, открывание, наличник, особенности
- Минимум вводов
- Не нарушает основной паттерн массового замера
- Генерация мини-отчёта (Excel/PDF)

## 3. Логика обработки измерений

### 3.1. Полная цепочка (Pipeline)

```
BLE (рулетка)
  ↓
Measurement (сырое значение + timestamp)
  ↓
ScenarioEngine (определяет: ширина/высота, номер в блоке)
  ↓
StabilityAnalyzer (разброс, стабильность между дверями)
  ↓
AdaptiveScenario (3×3 → 2×2 → 1×1 по стабильности)
  ↓
HeightAnalyzer (кластеризация высот → Δ-анализ → ДП/ДСТ/ДЧП)
  ↓
SignatureBuilder (генерирует подпись из контекста + справочников)
  ↓
Storage (SQLite + JSON)
  ↓
Export (Excel с подписями, датами, типами)
```

### 3.2. ScenarioEngine — управление сценарием замера

**Проблема:** замерщик делает 3 замера ширины, 3 высоты, потом может быть четверть и т.д. Система должна понимать, где он в процессе.

**Решение:** гибкий сценарий с допусками, не жёсткий шаблон.

```python
class Scenario:
    steps = [
        Step(type=WIDTH,  count=2..4, subtype=MAIN),      # Ширина основная
        Step(type=HEIGHT, count=2..3, subtype=MAIN),      # Высота основная
        Step(type=HEIGHT, count=1..3, subtype=QUARTER),   # Высота (четверть/доп)
    ]
```

**Типы отклонений:**
| Уровень | Что | Реакция |
|---------|-----|---------|
| 🟢 Нормальное | +1 замер, чуть другой порядок | Предупреждение |
| 🟡 Подозрительное | Лишний блок, слишком много значений | Запрос подтверждения |
| 🔴 Критическое | Ширина вместо высоты, значение вне физики | Стоп сценария |

### 3.3. StabilityAnalyzer + AdaptiveScenario

**Идея:** замерщик начинает с 3×3 (макс. точность), если видит стабильность — переходит на 2×2, потом на 1×1.

```python
enum PrecisionLevel:
    HIGH   # 3×3 — старт, нестабильные объекты
    MEDIUM # 2×2 — стабильные проёмы
    LOW    # 1×1 — только если нет четвертей и стабильная геометрия
```

**Правила переключения:**

⬇️ **Снижение точности** (ускорение):
- Разброс < 10мм
- Нет ошибок сценария
- 5+ дверей подряд стабильны
→ Предложение: "Перейти на 2×2? [Да] [Оставить 3×3]"

⬆️ **Повышение точности** (страховка):
- Разброс > 20мм
- Новый тип высоты
- Ошибки в сценарии
→ Автоматический возврат к 3×3

**Раздельная точность:** ширина может быть 2×, высота 3× — независимо.

### 3.4. HeightAnalyzer — определение типа высоты

**Проблема:** на одной двери могут быть несколько типов высоты (ДП, ДСТ, ДЧП). Их диапазоны пересекаются:
- ДП = 2180 ± 20 (до ЖБ плиты)
- ДСТ = 2100 ± 20 (до стяжки)
- ДЧП = 2120 ± 20 (до чистового)

**Решение: кластеризация + Δ-анализ**

```python
# 1. Кластеризация по близости значений
[2180, 2182, 2100, 2098, 2120, 2118]
  ↓
[2180, 2182]  → кластер 1
[2100, 2098]  → кластер 2
[2120, 2118]  → кластер 3

# 2. Анализ разниц (Δ) относительно базового (минимального)
base = 2100 (ДСТ)
2120 - 2100 = 20мм → Δ 10-30 → ДЧП (чистовой слой)
2180 - 2100 = 80мм → Δ 50-120 → ДП (конструкция)

# 3. Перенос паттерна между этажами
5 этаж: ДСТ + ДЧП
6-8 этаж: тот же паттерн → автоприменение
```

**Когда нужен ручной ввод:**
- Δ ≈ 30-50мм (серая зона)
- Только один замер высоты
- Пользователь выбирает ОДИН РАЗ → система запоминает

### 3.5. SignatureBuilder — генерация подписей

**Пример итоговой подписи:**
```
Д-31 П1.1.Кв3 Ст+14/16 Пр Сн Мнт Шт-
```

**Подпись = результат вычисления, не источник:**

```python
class SignatureBuilder:
    def build(self, measurement, context):
        parts = []
        
        # Контекст двери
        parts += context.door_mark          # Д-31
        parts += context.section_floor       # П1.1
        parts += context.apartment           # Кв3
        
        # Параметры из справочников
        parts += context.floor_type          # Ст+14/16
        parts += context.opening             # Пр
        parts += context.casing              # Сн
        parts += context.wall_material       # Мнт
        parts += context.plaster             # Шт-
        
        # Тип измерения
        parts += measurement.type_label      # Измерение / Дст / Дчп / Четверть
        
        return ' '.join(parts)
```

**Иерархия источников (приоритет):**
1. **Ручной override** (замерщик исправил) — высший приоритет
2. **Индивидуальные особенности двери** (сдвиг проёма, укороченный наличник)
3. **Данные из PDF** (если есть разметка)
4. **Типовая планировка** (справочник для объекта)
5. **Глобальные настройки проекта** (по умолчанию)

```python
def resolve(key):
    return (override.get(key) 
         or individual.get(key) 
         or pdf_data.get(key) 
         or typical.get(key) 
         or global_default.get(key))
```

**Наследование параметров:**
- Если тип высоты совпадает на предыдущей двери → автоприменение к следующей
- Если несколько дверей подряд с одинаковыми параметрами → не вводить заново

### 4.1. Типовые параметры двери

### 6.2. BLE протокол Bosch GLM 50C

**UUID сервиса:** `02A6C0D0-0451-4000-B000-FB3210111989` (MIRACULIX_SERVICE)

**Формат пакета:**
```
C0 55 10 06 XX XX XX XX  → расстояние (float32 LE, метры)
```

**Команды:**
| Команда | Hex | Описание |
|---------|-----|----------|
| Измерение | `C0 40 00 EE` | Запуск измерения |
| Лазер ON | `C0 41 00 96` | Лазер ВКЛ |
| Лазер OFF | `C0 42 00 1E` | Лазер ВЫКЛ |
| BLE init | `C0 55 02 01` | Инициализация BLE |
| BLE cmd | `C0 56 01 00 1E` | Лазер + замер |

## 9. Python Engine (будущий)

### 9.1. Назначение

1. **Быстрый пересчёт:**替代 357K формул Excel → расчёт за 2-5 секунд
2. **Валидация:** сравнение результата с текущей Обработка.xlsx
3. **Генерация отчётов:** создание выходных файлов без Excel
4. **Аналитика:** статистика по объектам, поиск паттернов

### 9.2. Модули

```python
# main.py — оркестрация
engine = ProcessingEngine()
result = engine.process(edit_file, output_file)

# parser.py — чтение edit.xlsx
raw = DataParser.read(edit_file)

# scenario.py — сценарий замера
scenario = ScenarioEngine.detect(raw)

# height.py — анализ высот
clusters = HeightAnalyzer.cluster(raw.heights)
types = HeightAnalyzer.detect_types(clusters)

# signature.py — генерация подписей
signatures = SignatureBuilder.build(doors, context, dictionaries)

# export.py — генерация Excel
exporter = ExcelExporter(result, signatures)
exporter.save(output_file)
```

### 9.3. Переходный период

Python и Excel работают **параллельно**:
- Python → быстрый результат
- Excel → страховка для ручных правок
- Сверка → если совпадает, переходим на Python
