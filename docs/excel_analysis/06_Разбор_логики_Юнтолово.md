## 1. Исходный файл (edit.xlsx)

### 1.1 Список измерений

**Размер:** A1:AN1308, 1308 строк, 6 заполненных колонок.

Это **сырой экспорт** с прибора Bosch GLM. Каждая строка — одно линейное измерение.

| Столбец | Заголовок | Пример |
|---------|-----------|--------|
| A | Название | "Измерение" |
| B | Значение | "2169,4" |
| C | Единица | "mm" |
| D | Дата | "07/04/26" |
| E | Время | "18:09" |
| F | Вид | "Длина" |

**Ключевая особенность:** дверь НЕ идентифицирована в этом листе. Маркеры дверей (типа `П1.22.Кв1 Сн Шт-`) появляются как текстовые значения в столбце A на определённых строках (например, строка 67, 133, 199). Между маркерами идут группы "Измерение" — это 3 ширины + 3 высоты одной двери.

**Структура данных:**
```
Строка 2-66:  Дверь 1 (Кв1, секция 1, этаж 1) — 65 измерений
Строка 67:    Маркер "П1.22.Кв1 Сн Шт-" (начало новой двери)
Строка 68-132: Дверь 2 — 65 измерений
...
```

Каждая дверь = ~65 строк: 3 замера ширины (Ш1, Ш2, Ш3) + 3 замера высоты (В1, В2, В3) + промежуточные замеры (четверти, зубы, стяжки и т.д.).

### 1.2 Спецификация заполнения дверей

Проектные параметры каждой марки двери. Ключевые столбцы:

| Столбец | Заголовок | Пример (строка 2) |
|---------|-----------|-------------------|
| D | Высота проема | 2100 |
| E | Ширина проема | 1010 |
| F | Открывание | правое |

## 2. Обработка.xlsx — постраничный разбор

Файл содержит **17 листов**, организованных в конвейер от сырых данных к финальным отчётам. Именованные диапазоны ссылаются на `edit.xlsx` через префикс `[1]`.

### 2.1 Список измерений фильтр

**Назначение:** Первичный приём сырых данных из `edit.xlsx`. Копирует все строки `Список измерений` с формулами `IF(ISBLANK(...), "", '[1]Список измерений'!X)`.

**Откуда:** `[1]Список измерений` (edit.xlsx, лист "Список измерений")

**Куда:** Автомаркировка, Список фильтр

**Ключевые формулы:**
```excel
=IF(ISBLANK('[1]Список измерений'!A2),"",'[1]Список измерений'!A2)
=IFERROR(LARGE(B:B,ROW(B1)),"")     — нумерация
=IFERROR(INDEX(C:C,MATCH($I1,$B:$B,0)),"")  — индексация по номеру
```

**Активные столбцы:** 23 из 33. Столбцы A-H — сырые данные, I-O — индексированные, P-V — данные из Автомаркировки.

### 2.2 Автомаркировка

**Назначение:** **Распознавание дверей** — главный лист классификации. Превращает сырые строки "Измерение" в структурированные двери, определяя:
- Какой набор измерений принадлежит какой двери
- Тип проёма (ДП/ДСТ/ДЧП)
- Маркировку
- Границы диапазонов для усреднения

**Откуда:** `Список измерений фильтр`, именованные диапазоны из edit.xlsx

**Куда:** `Список измерений обработаный`, `Маркировка`, `Список фильтр`

**Ключевые формулы:**

1. **Определение начала новой двери (смена маркировки):**
```excel
=IF(B2=E2,1,0)   — столбец D: флаг смены группы
```

2. **Классификация типа замера:**
```excel
=IF(COUNTIF(Замер_словарьСКР,E2)>0,"Change",
  IF(ISNUMBER(SEARCH("Закрыто",E2)),"Change",
  IF(ISNUMBER(SEARCH("Нетпроема",E2)),"Change",...)))
```
Столбец Q — статус "Change" означает переход к новой двери.

3. **Медиана измерений двери:**
```excel
=MEDIAN(INDIRECT($W2))    — столбец O
```
где W2 — динамический диапазон вида `N2:N4`.

4. **Округление до 5мм:**
```excel
=IF(ISNUMBER(SEARCH("Лед+",Q2)),SUBSTITUTE(Q2,"Лед+",""),MROUND(O2,5))
```

5. **Определение типа двери по словарю:**
```excel
=TRIM(IF(E2="0 П00","0 П00",
  IF(AT2<>"",BH2,
  IF(AI2<>"",AI2,E2))))
```

**Активные столбцы:** 79 из 191. Ключевые: A(№), B(Название), H(Очищенное название), N(Значение числовое), O(Округленное), Q(Change-флаг), R(Тип замера), W(Диапазон усреднения).

### 2.3 Список измерений обработаный

**Назначение:** **Промежуточная обработка** — извлечение из Автомаркировки конкретных значений замеров, привязанных к дверям.

**Откуда:** `Автомаркировка`

**Куда:** `Расчет`, `Медиана и список`

**Ключевые формулы:**
```excel
=TRIM(Автомаркировка!N2)    — F: Значение числовое
=TRIM(Автомаркировка!O2)    — G: Округленное значение
=IFERROR(IF(ISNUMBER(SEARCH("Лед+",B4)),SUBSTITUTE(B4,"Лед+",""),MROUND(G4,5)),"")  — H: Фактический проем
=TRIM(Автомаркировка!Q2)    — I: Формат замера
```

**Активные столбцы:** 78 из 101. Включают ширину проёма, четверти, высоты (ДП, ДСТ, ДЧП), примечания.

### 2.6 Расчет

**Назначение:** **Главный вычислительный лист** — здесь происходит вся бизнес-логика: расчёт отклонений, определение дефектов, формирование финальных размеров.

**Размер:** 500 строк, 223 столбца (самый широкий лист).

**Откуда:** `Список измерений обработаный`, `Словарь условий` (через именованные диапазоны)

**Куда:** `Замер`, `Дефектная ведомость`, `Минимум`, `Новый шаблон`, `Шахматка`

**Ключевые столбцы (группировка):**

| Диапазон | Назначение |
|----------|-----------|
| A-D | Идентификация двери |
| E-O | Размеры проёма (ширина, высоты до разных уровней) |
| P-AY | Четверти, зубы, кровь, коммуникации, ступени |
| AZ-BM | За проемом / в проеме параметры |
| BN-DF | Рама, доборы, пороги |
| DG-GL | Отклонения, дефекты |

**Ключевые формулы:**

1. **Отклонение по высоте (столбец GL):**
```excel
=IF(AND(DA3='[1]Словарь условий'!$AX$2, DB3='[1]Словарь условий'!$AY$2, BC3<>""), BC3,
  IF(AND(DA3='[1]Словарь условий'!$AX$2, DB3='[1]Словарь условий'!$AY$3, BG3<>""), BG3,
  IF(AND(DD3=...))))
```
Многоуровневая проверка: сравнивает фактический размер с проектным, используя допуски из Словаря условий.

2. **Отклонение по ширине (столбец GM):**
```excel
=IF(DC3='[1]Словарь условий'!$BA$2, AU3,
  IF(DC3='[1]Словарь условий'!$BA$3, AV3,
  IF(DC3='[1]Словарь условий'!$BA$4, CY3, "")))
```

3. **Ширина проёма (столбец E):**
```excel
=INDEX('Список измерений обработаный'!P:P, ...)
```

**Назначение:** **Финальная агрегация** — собирает все данные из `Расчет` и `Маркировка` в единую таблицу замеров. Это основной лист, который экспортируется в выходной файл.

Приоритет: прямое измерение > через чистовое + покрытие > через плиту - покрытие.

## 3. Трассировка конкретных дверей

### 3.1 Дверь типовая — Кв1, Дверь 1 (Марка 31, правое открывание)

## 5. Бизнес-правила (логика над формулами)

### 5.1 Как определяется тип проёма (ДП/ДСТ/ДЧП)

**Типы проёмов:**
| Код | Расшифровка | Измерение |
|-----|------------|-----------|
| **ДП** | До плиты | Высота замерена от пола-плиты до потолка |
| **ДСТ** | До стяжки | Высота замерена от стяжки до потолка |
| **ДЧП** | До чистового покрытия | Высота замерена от чистового пола |

**Механизм определения:**
1. В `Словарь условий` столбец C содержит коды: `дп`, `дст`, `уз`, `ду`
2. В `Автомаркировка` столбец Q (Change-флаг) проверяет:
   - Если `COUNTIF(Замер_словарьСКР, E2) > 0` → "Change" (новый тип замера)
   - Если содержит "Лед+", "Нетпроема", "Нетстены", "Незамер", "Неготов", "Закрыто", "Свежаястяжка" → "Change"
3. В `Список измерений обработаный` столбец I (Формат замера) определяет:
   - Если есть замер до плиты → "Дп"
   - Если есть замер до стяжки → "Дст"
   - Если есть замер до чистового → "Дчп"

### 5.2 Как рассчитываются отклонения

**Отклонение по высоте:**
```
Отклонение = Расчётная высота - Проектная высота
```
- Расчётная = Высота до плиты - Высота пола (из справочника "Полы")
- Проектная = из "Спецификация заполнения дверей" (2100мм)

**Текстовое описание дефекта:**
```excel
=IF(отклонение > 0, "Проем выше на " & отклонение & " мм",
 IF(отклонение < 0, "Проем ниже на " & ABS(отклонение) & " мм",
 ""))
```

**Отклонение по ширине:**
Аналогично: `Фактическая ширина - Проектная ширина`

**Допуски:** берутся из `Словарь условий` (столбцы AV, AW). Если отклонение в пределах допуска — дефект не фиксируется.

### 5.3 Как формируется финальная маркировка

**Полная маркировка** (столбец B в Маркировка):
```
{Марка} {Позиция}.{Секция}.{Этаж}.{Квартира} {Стяжка} {Открывание} {Наличник} {Стены} {Штукатурка}
```

**Пример:** `31 П1.1.Кв1 Ст-6.1/15.1 Пр Сн Скц/Мнт Шт`

Разбор:
- `31` — марка двери
- `П1.1.Кв1` — позиция: П1 (помещение тип 1), 1 (секция), 1 (этаж), Кв1 (квартира 1)
- `Ст-6.1/15.1` — стяжка: тип 6.1/15.1
- `Пр` — открывание правое
- `Сн` — с наличником
- `Скц/Мнт` — материал стен: камень стеновой / монолит
- `Шт` — штукатурка

**Краткая маркировка** (столбец C): только `31` или `31л` (л = левое открывание).

### 5.4 Как определяются дефекты

**Критерии дефекта:**

1. **Отклонение по высоте** — если расчётная высота != проектная (с учётом допуска):
   - "Проем выше на N мм"
   - "Проем ниже на N мм"

2. **Отклонение по ширине** — если фактическая ширина != проектная:
   - "Выступ справа N мм"
   - "Выступ слева N мм"

3. **Перепад полов** — если разница уровней пола перед/за проемом:
   - Записывается в "Дефектная перепад полов"

4. **Специальные состояния:**
   - "Закрыто" — проём не доступен для замера
   - "Нетпроема" — проём отсутствует
   - "Нетстены" — стена отсутствует
   - "Незамерен" — замер не выполнен
   - "Свежаястяжка" — стяжка ещё не готова
   - "Лед+" — наледь/конденсат

**Рекомендации:**
- При отклонении: "Привести проем в соответствии с проектом"
- При выступе: указывается сторона и размер

### 5.5 Логика расчёта размеров рамы

**Ширина рамы** = Фактическая ширина проёма - 40мм (монтажный зазор 20мм с каждой стороны)

**Высота рамы:**
- Если ДП (до плиты): `Высота - Высота_пола - 120мм` (порог + зазор)
- Если ДСТ (до стяжки): `Высота - 120мм` (уже учтена стяжка)

**Пример:** Дверь 1: 1010 х 2205 до плиты, пол 100мм
- Ширина рамы: 1010 - 40 = 970
- Высота рамы: 2205 - 100 - 25 = 2080
- Рама: **970х2080**
