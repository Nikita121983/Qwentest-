# План разработки приложения Bosch GLM 50C Reader

**Цель:** Создать минимальное рабочее приложение (MVP) на C# (.NET 8, Windows Forms) для подключения к рулетке, приёма измерений и экспорта данных.

---

## 1. Стек технологий

- **Язык:** C# 12+
- **Фреймворк:** .NET 8
- **UI:** Windows Forms (простота и скорость)
- **BLE:** `Windows.Devices.Bluetooth` (WinRT) через NuGet пакет `Microsoft.Windows.CsWinRT` или стандартные ссылки в .NET 8 для Windows.

---

## 2. Архитектура

```text
GlmReader/
├── Core/
│   ├── BleManager.cs          # Управление BLE (Scan, Connect, Subscribe)
│   └── GlmProtocol.cs         # Парсинг пакетов, генерация команд
├── Models/
│   └── Measurement.cs         # Модель данных (Result, A, B, Angle, Type)
├── UI/
│   └── MainForm.cs            # Основной интерфейс
└── Program.cs
```

## 4. Этапы реализации (для AI)

### Этап 1: Скелет и UI
- Создать проект WinForms.
- Разместить элементы управления (ComboBox, Buttons, Labels, DataGridView).
- Настроить верстку.

### Этап 2: BLE Менеджер
- Реализовать сканирование устройств (`BluetoothLEAdvertisementWatcher`).
- Реализовать подключение (`BluetoothLEDevice.FromBluetoothAddressAsync`).
- Получить сервис и характеристику.
- Реализовать запись команд (WriteValueAsync).

### Этап 3: Протокол и Парсинг
- Создать класс `GlmProtocol`.
- Реализовать метод `ParsePacket(byte[] data)`.
- Обработать типы: `0x06` (Direct), `0x2A/0x2E` (Indirect), `0x36` (Double).
- **Важно:** Для типа `0x36` компонент `a` не передается — использовать значение из предыдущего пакета `0x06`.

### Этап 4: Интеграция
- Связать `BleManager` с UI.
- При получении данных обновлять UI (через `Invoke`).
- Добавить обработку ошибок (отключение, таймауты).

### Этап 5: История и Экспорт
- Реализовать кнопку "Синхронизация".
- Реализовать сохранение `List<Measurement>` в CSV файл.

## 5. Известные проблемы / Вопросы

1. **Точка отсчёта:** Неизвестно, где передается флаг (Задняя/Штатив/Передняя).
   - *Решение:* Вывести "сырые" байты Tail (19-30) в лог для отладки при переключении.
2. **Библиотека BLE:** В .NET 8 для Windows лучше использовать `Windows.Devices.Bluetooth` напрямую, чтобы не тянуть лишние зависимости, если это возможно.
