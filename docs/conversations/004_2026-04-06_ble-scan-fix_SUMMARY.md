## 📋 Краткое содержание

Диагностика и исправление бага: APK собирался успешно, но при запуске не находил устройства BLE при сканировании.

---

## 🔍 Диагностика

### Проблема
- Сборка APK проходит успешно ✅
- Приложение запускается и показывает UI ✅
- Сканирование запускается ("Сканирование...") ✅
- **НО** список устройств пустой ❌
- Nordic nRF Connect успешно видит и подключается к рулетке ✅

### Корень проблемы
В `MeasurementListActivity.kt`, метод `setupBleCallbacks()`:

```kotlin
bleManager.onDeviceFound = { mac, name ->
    bleManager.stopScan()    // ← ОСТАНАВЛИВАЕТ сканирование
    bleManager.connect(mac)  // ← ПОДКЛЮЧАЕТСЯ автоматически
}
```

Этот callback:
1. Срабатывал на **первое найденное BLE устройство**
2. Немедленно останавливал сканирование
3. Пытался подключиться автоматически

Это мешало диалогу `showBleDialog()` заполнять список устройств для выбора пользователем.

---

## 🔑 Ключевые решения

| Решение | Обоснование |
|---------|-------------|
| Убрать автоподключение в `setupBleCallbacks()` | Пользователь должен выбирать устройство в диалоге |
| Корректно управлять callback'ами в диалоге | Диалог устанавливает свой callback для заполнения списка |
| Не сбрасывать callback при закрытии диалога | Восстанавливать logging-only callback, а не null |
