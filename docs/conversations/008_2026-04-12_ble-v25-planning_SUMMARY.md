## Хронология работ

### 1. Диагностика BLE (начало сессии)
- Рулетка подключается, но измерения не приходят в список
- При переключении точки отсчёта добавляются нулевые измерения (devMode=62)
- Приложение крашится при входе в список измерений (ClassCastException — не указан android:name)
- **Причина:** GlmBleManager НЕ извлекает refEdge/devMode из пакетов. Ответы на Set Ref (devMode=62) НЕ фильтруются и попадают в БД.

### 2. Анализ исходного кода MM/MO
- MM GLMDeviceController.handleEDCMessage() — извлекает refEdge/devMode, НЕ сохраняет ответы на команды в историю
- MO GLMDeviceController.onEvent() — аналогично
- Наш код: состояние объявлено (currentRefEdge, currentMeasurementType) но НИКОГДА не обновляется

### 3. Мокап v25
- Создан `mockup_ui_v25.html` — финальный UI с flip анимациями, динамическими цветами, 4 точками отсчёта, 12 типами
- Toolbar: 4 цвета (rear=#2B5878, front=#2e7d32, tripod=#c62828, pin=#FF9800)
- Фон списка: 12 цветов по типу (средние из v21)
- Тёмная тема: инвертированные цвета

### 4. Debug окно — новые требования

- Кнопка Clear → Save/Open Log
- Расшифровка протокола в логе (Пр: 1033,1 мм)
- Показ refEdge и measurementType
- Кнопки Set Ref (0-3) / Set Type (0-26)
- Импорт таблицы значений для неизвестного протокола

### 5. Кастомные цветовые схемы

### 6. Кастомные иконки

## Итоговые решения
1. **Фильтр devMode=62** — НЕ сохранять ответы на Set Ref как измерения
2. **Извлечение refEdge/devMode** — из КАЖДОГО пакета в GlmBleManager.onPacketReceived()
3. **UI соответствует v25** — toolbar по точке отсчёта, фон списка по типу, flip анимации
4. **Все задачи Beads** содержат ссылки на исходники MM/MO для сопоставления ПЕРЕД кодом
