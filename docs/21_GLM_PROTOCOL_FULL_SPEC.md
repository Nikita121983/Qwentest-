# Bosch GLM 50C — Полная спецификация BLE протокола

**Дата:** 3 апреля 2026 г.
**Статус:** ✅ Подтверждено реверс-инжинирингом (3 HCI лога, сопоставление с TXT экспортом).

---

## 1. Подключение (BLE GATT)

| Параметр | Значение |
|----------|----------|
| **Service UUID** | `02A6C0D0-0451-4000-B000-FB3210111989` |
| **Characteristic UUID** | `02A6C0D1-0451-4000-B000-FB3210111989` |
| **Тип соединения** | BLE (Bluetooth Low Energy) |
| **Имя устройства** | `GLM 50 C` (или похожее) |

---

## 2. Команды управления (Host → Device)

Команды отправляются в характеристику `02A6C0D1...` с ответом (Write With Response).

| Команда | Hex | Описание |
|---------|-----|----------|
| **Init** | `C0 55 02 01 00 1A AC` | Инициализация соединения (шлется при подключении) |
| **Measure** | `C0 56 01 00 1E AC` | Запуск измерения (вкл лазер + замер) |
| **Sync History** | `C0 55 10 3A 09 1A 71 00 00 00 00 78` | Запрос истории (рулетка вышлет последние 30 замеров) |

---

## 3. Формат пакета данных (Device → Host)

Все данные приходят в формате уведомлений (Notifications).

```
C0 55 10 TT SS NN 71 [Data Payload] [Tail]
│  │  │  │  │  │  │   └─ Данные (float32)
│  │  │  │  │  │  └─ 0x71 (Constant)
│  │  │  │  │  └─ NN: Sequence Number (растет с каждым пакетом)
│  │  │  │  └─ SS: Status (0x08 = Result Ready)
│  │  │  └─ TT: Type (Тип измерения)
│  │  └─ 0x10 (Data Prefix)
│  └─ 0x55 (BLE Prefix)
└─ 0xC0 (Start Byte)
```

### 3.1. Типы измерений (TT)

| Hex | Тип | Единица | Формат данных |
|-----|-----|---------|---------------|
| `0x06` | **Прямой замер** | мм | `float32` (метры) |
| `0x2A` | **Косвенная высота** | мм | `Result` + `Comp_a` + `Angle_b` |
| `0x2E` | **Косвенная длина** | мм | `Result` + `Comp_a` + `Angle_b` |
| `0x36` | **Двойная косвенная** | мм | `Result` + `Comp_b` + `Angle_c` |
| `0x3E` | **Площадь** | дм² | `float32` (значение × 1000 = мм²) |
| `0x1E` | **Объём** | дм³ | `float32` (значение × 10⁶ = мм³) |
| `0x21` | **Угол** | мград | `float32` (миллиградусы) |
| `0x0A` | **Непрерывный** | мм | `float32` (серия значений) |
| `0xF2` | **Референс** | мм | `float32` (тестовые значения) |

### 3.2. Структура данных (Payload)

Для косвенных измерений (`0x2A`, `0x2E`, `0x36`):

| Смещение | Размер | Описание |
|----------|--------|----------|
| **0-6** | 7 байт | Заголовок (`C0 55 10 TT SS NN 71`) |
| **7-10** | 4 байта | **Результат** (float32 Little Endian, метры) |
| **11-14** | 4 байта | **Компонент** (float32, метры). Для `0x36` это `b`, для `0x2A/0x2E` это `a`. |
| **15-18** | 4 байта | **Угол** (float32, градусы). |
| **19-30** | 12 байт | **Tail** (Метаданные: ID, счетчики). Точная структура не известна. |

> **Важно:** Компонент `a` для режима `0x36` (Двойная косвенная) **не передается** в пакете. Приложение должно сохранять значение предыдущего прямого замера как `a`.

---

## 4. Heartbeat / Статус (Device → Host)

Периодические пакеты состояния.

```
C0 55 02 F1 [Mode] [Angle] [Checksum]
```

| Байт | Описание |
|------|----------|
| **0-3** | `C0 55 02 F1` (Заголовок) |
| **4** | **Mode** (Режим рулетки: 0-15). Соответствует типам измерений. |
| **5** | **Angle** (Текущий угол наклона, 0-255). |
| **6** | **Checksum** (Почти всегда `0xAC`). |

---

## 5. Точка отсчёта (Reference Point)

**Статус:** ❓ **Не расшифровано.**
Рулетка имеет 3 точки отсчёта: Задняя грань, Ось штатива, Передняя грань.
**Гипотеза:** Рулетка отправляет уже исправленное значение (с учетом смещения). Флаг точки отсчёта, вероятно, находится в **Tail** (байты 19-30) или в **Heartbeat Mode**.
**План:** Выяснить экспериментально при разработке приложения.

---

## 6. Примеры парсинга

### Прямой замер (2078.2 мм)
`C0 55 10 06 08 08 71 69 00 05 40 00 ...`
- Type: `06` (Direct)
- Value (bytes 7-10): `69 00 05 40` → `2.0782` м → **2078.2 мм**

### Косвенная длина (1197.9 мм, a=1884.2, b=50.2°)
`C0 55 10 2E 08 36 71 DD 54 99 3F 77 2D F1 3F 69 BC 48 42 ...`
- Type: `2E` (Indirect Length)
- Result (7-10): `DD 54 99 3F` → `1.1979` м → **1197.9 мм**
- Comp_a (11-14): `77 2D F1 3F` → `1.8842` м → **1884.2 мм**
- Angle (15-18): `69 BC 48 42` → `50.2`°
