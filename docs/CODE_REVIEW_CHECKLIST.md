# Чеклист код-ревью

Проверь код перед отправкой на ревью или перед мержем.

---

## ✅ Код

- [ ] Код следует правилам из `CODE_RULES.md`
- [ ] Файлы ≤400 строк (критично: >500)
- [ ] Классы ≤250 строк (критично: >300)
- [ ] Функции ≤50 строк (критично: >100)
- [ ] Параметры функции ≤5 (критично: >7)
- [ ] Нет дублирования кода (DRY)
- [ ] Нет закомментированного кода
- [ ] Нет `print()` (используется `logging`)
- [ ] Переменные/функции названы понятно
- [ ] Нет "магических чисел" (вынесены в константы)

---

## ✅ Тесты

- [ ] Тесты написаны для нового кода
- [ ] Все тесты проходят: `pytest tests/ -v`
- [ ] Покрытие ≥80%: `pytest --cov=src`
- [ ] Тесты на граничные случаи
- [ ] Тесты на ошибки (исключения)
- [ ] Названия тестов понятные

---

## ✅ Линтеры

- [ ] ruff проходит: `ruff check src/`
- [ ] black проходит: `black --check src/`
- [ ] mypy проходит: `mypy src/`
- [ ] Pre-commit хуки установлены и работают

---

## ✅ Документация

- [ ] Docstring на всех публичных функциях
- [ ] Docstring в формате Google style
- [ ] README.md обновлён (если изменился API)
- [ ] CHANGELOG.md обновлён
- [ ] Type hints на всех функциях

---

## ✅ Архитектура

- [ ] Зависимости только вниз (GUI → IO → Core → Utils)
- [ ] Нет круговых зависимостей
- [ ] Классы следуют SOLID
- [ ] Нет утечек абстракций

---

## ✅ Безопасность

- [ ] Нет хардкодных паролей/ключей
- [ ] Нет чувствительных данных в логах
- [ ] Валидация входных данных
- [ ] Обработка исключений корректная

---

## ✅ Git

- [ ] Коммиты атомарные (один коммит = одно изменение)
- [ ] Сообщения коммитов понятные
- [ ] Ветка названа правильно (feature/xxx, fix/xxx)
- [ ] Нет конфликтов слияния
- [ ] PR описывает изменения
