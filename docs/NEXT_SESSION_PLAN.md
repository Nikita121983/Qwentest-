## 1. Приоритетные задачи

### 1.1. Реорганизация папки (ПЕРВОЕ ДЕЙСТВИЕ)
**Файл:** `docs/REORGANIZATION_PLAN.md`
- Выполнить шаги 1-4 из плана (создание папок, перемещение логов/багрепортов/скриптов)
- Выполнить шаг 5 (удаление временных файлов)
- Обновить `.gitignore`
- Удалить `FOLDER_STRUCTURE_TEMP.txt`

### 1.2. База данных и структура проектов
**Контекст:** Переходим от простого списка замеров к системе "Проекты → Замеры".
- Создать `ProjectEntity` и `ProjectDao` в Room
- Добавить связь `Project (1) → Measurement (N)`
- Реализовать логику создания папки проекта на диске (с чекбоксом в диалоге)
- Настроить параллельную запись в `measurements_log.json` внутри папки проекта (JSON бэкап)

### 1.3. Экран списка проектов
- Создать `ProjectListActivity` с `RecyclerView`
- Дизайн карточек как в MM (название, дата, кол-во замеров)
- Диалог создания проекта:
  - Поле ввода имени
  - Чекбокс `[x] Создать папку на диске` (по умолчанию ВКЛ)
  - Выбор корневого каталога (через `Storage Access Framework` или хардкод для прототипа)

### 1.4. Экран списка измерений (внутри проекта)
- Обновить `MeasurementListActivity` для работы с `project_id`
- Добавить сворачиваемую панель Remote Control (стиль MM):
  - Кнопки выбора типа замера и точки отсчета
  - Кнопка "Включить" (Лазер)
  - Анимация сворачивания/разворачивания
- Реализовать Smart Scroll (автопрокрутка только если пользователь внизу списка)
- Добавить FAB кнопки навигации (⬆ В начало, ⬇ В конец)

### 1.5. Режим выделения и удаление
- Долгое нажатие → режим выделения (чекбоксы)
- Верхняя панель: `[Интервал] [Все] [Удалить] [Отмена]`
- Логика выделения интервала (тап на первый → тап на последний)
- Диалог подтверждения удаления

### 1.6. Ввод подписей (Умные функции)
- `InputFilter` для авто-капитализации (Smart Mode: `д-11` → `Д-11`)
- Сохранение при потере фокуса (без диалога "ОК")
- Позиция курсора: в начале для стандартных подписей, в конце для редактируемых
- История версий (3 слота): `caption_1`, `caption_2`, `caption_3`
- Кнопка "История" в режиме выделения для отката

## 2. Технические детали

### 2.1. Файловая структура проекта
```
/{Выбранный_Корень}/{Имя_Проекта}/
├── measurements_log.json    # JSON бэкап (параллельно с SQLite)
├── media/                   # Будущие медиафайлы
└── exports/                 # Экспорты XLSX/CSV
```

### 2.2. Экспорт
- Выбор формата: `.xlsx` / `.csv`
- Выбор колонок (галочки) или "Полная копия БД"
- Библиотека: Apache POI (уже подключена)

### 2.3. Ресурсы (из APK MM)
Скопировать из `Android/.gitignore-files/sources/decompiled-full/resources/res/`:
- `drawable/ic_measuremode_*.png` — иконки типов замеров
- `drawable/ic_remote_measure.png` — иконка дистанционного управления
- `drawable/ic_menu_modul_measure_*.png` — иконки режимов
