## 🎯 Главная задача: Протестировать исправление BLE сканирования

### Что было исправлено
**Проблема:** Приложение не находило устройства при сканировании, хотя Nordic nRF Connect видел их нормально.

**Корень:** Callback `onDeviceFound` в `setupBleCallbacks()` автоматически:
1. Останавливал сканирование при первом найденном устройстве
2. Пытался подключиться к нему

Это мешало диалогу `showBleDialog()` заполнять список устройств.

**Решение (коммит 4736c1f):**
- Убрали автоподключение в `setupBleCallbacks()` — теперь callback только логирует
- В диалоге корректно устанавливаем callback для заполнения списка
- При закрытии диалога восстанавливаем logging-only callback, а не сбрасываем в null

### Что нужно сделать
1. **Собрать APK:**
   ```bash
   cd GlmReaderAndroid
   gradlew.bat assembleDebug
   ```
   APK будет в: `GlmReaderAndroid/app/build/outputs/apk/debug/app-debug.apk`

2. **Установить на телефон** и протестировать:
   - Открыть приложение
   - Нажать иконку компаса (подключение)
   - Нажать "Сканировать"
   - **Ожидание:** В списке должны появиться устройства, включая `Bosch GLM50C x9370` (MAC: `00:13:43:D5:21:5D`) и/или `x5747`
   - Выбрать устройство → "Подключить"
   - Проверить подключение

3. **Смотреть логи через logcat:**
   ```bash
   adb logcat | findstr "BLE"
   ```
   Искать:
   - `BLE_SCAN: Found:` — все найденные устройства
   - `BLE_SCAN: >>> BOSCH DEVICE FOUND:` — конкретно Bosch
   - `BLE_DIALOG: Found in dialog:` — устройства в диалоге
   - `BLE: onConnectionStateChange:` — статус подключения

### Если всё работает
- Закрыть задачу Qwentest-ai7 (уже закрыта ✅)
- Перейти к следующей задаче: получение данных с рулетки

- Убедиться, что Bluetooth включен и разрешения выданы
- Проверить, что рулетка включена и видна (Nordic nRF Connect должен видеть)
