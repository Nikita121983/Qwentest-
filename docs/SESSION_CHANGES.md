## 1. Критические исправления Протокола (EDC Protocol)

### 1.1. Формула байта заголовка (EDC Header)
**Проблема:** Ранее мы формировали байт заголовка, считая `syncControl` старшим битом (значение `0x40` для инициализации). Связь с рулеткой была нестабильной или отсутствовала.
**Анализ:** При декомпиляции `EDCFrameFactory.java` и `Headers` (BitField) из исходников Measuring Master выяснилось, что поля упаковываются в битовое поле последовательно:
1.  `syncControl` (1 бит)
2.  `keypadBypass` (1 бит)
3.  `devMode` (6 битов)
**Решение:** Изменена формула расчета на `(devMode << 2) | (keypadBypass << 1) | syncControl`.
**Результат:**
*   Init (AutoSync ON) теперь отправляет `0x01` (вместо `0x40`).
*   Смена режима (devMode=60) использует корректные маски.
*   Смена точки (devMode=62) использует корректные маски.

### 1.2. Маппинг режимов (SyncMode -> EDCMode)
**Проблема:** Приложение отправляло индексы режимов из UI напрямую, что не совпадало с протоколом.
**Анализ:** Изучен метод `turnSyncModeToEDCMode` в `EDCInputMessage.java`.
**Решение:** Реализована функция `setMeasurementType(syncMode)`, которая преобразует UI-индекс в EDC-значение перед отправкой.

### 1.3. Точки отсчёта (Reference Edge)
**Проблема:** Несоответствие порядка точек в UI и протоколе (Front/Rear/Tripod).
**Анализ:** Константы в `EDCInputMessage.java`: `FRONT=0`, `TRIPOD=1`, `REAR=2`, `PIN=3`.
**Решение:**
*   Исправлены массивы названий и иконок в `MeasurementListActivity`.
*   Добавлена логика переключения кубика с учетом `Pin` (опционально через настройки).

## 2. Логика кнопки "Включить" (Лазер)

**Требование:** Кнопка должна работать как фиксатор (тумблер):
1.  Нажатие -> Включить лазер (прицеливание), замера нет.
2.  Повторное нажатие -> Сделать замер, выключить лазер.

**Реализация:**
*   Добавлен флаг `isLaserOn` в `GlmBleManager`.
*   **Команда "Вкл"**: Отправляется пакет `Laser On` (`devMode=0`, `remoteCtrlData=1`, Header `0x01`).
*   **Команда "Выкл/Замер"**: Отправляется пакет `Trigger` (`devMode=0`, `remoteCtrlData=0`, Header `0x01`).

## 3. Исправление "Зависания" SendThread (Проблема двойного нажатия)

**Проблема:** Если нажать кнопку смены типа/точки сразу после замера, пока рулетка занята, команда не проходила. Иногда требовалось нажимать дважды.
**Причина:** В цикле `startSendThread` при таймауте ожидания `MASTER_READY` вызывался `break`, который убивал поток отправки.
**Решение:**
*   Заменен `break` на `continue`. Теперь при таймауте команда сбрасывается, но поток продолжает работать.
*   Добавлена пауза 500мс после таймаута.

## 4. UI/UX Исправления

### 4.1. Состояние кнопок при запуске
**Проблема:** При входе в список измерений, если автоподключение уже сработало, кнопки оставались серыми (disabled).
**Решение:** В `setupBleCallbacks` добавлена принудительная проверка `bleManager.isConnected` сразу после инициализации.

### 4.2. Переключатель "Показывать Pin"
**Проблема:** Не все рулетки поддерживают точку отсчета "Pin" (EDC 3).
**Решение:**
*   Добавлен `Switch` в Настройки.
*   Логика кубика проверяет этот флаг.
