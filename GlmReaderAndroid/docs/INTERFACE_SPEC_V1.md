# GlmReaderAndroid — Спецификация Интерфейса и Логики (v1.0)

**Дата:** 4 апреля 2026 г.
**База:** Measuring Master (MM) + требования пользователя.

---

## 1. Общие принципы
*   **Стиль:** Прототип визуально повторяет **Bosch Measuring Master** (цвета, иконки, структура).
*   **Код:** Код управления (Remote Control) из MM используется **только как референс** (HEX-команды). Логика пишется с нуля на Kotlin/Coroutines для стабильности.
*   **Хранение:** SQLite (Room). Никаких JSON-файлов для критичных данных.

---

## 2. Структура приложения
1.  **Список Проектов:** Карточки проектов (Название, Дата, Кол-во замеров).
2.  **Список Измерений (Основной экран):** Список замеров внутри выбранного проекта + Панель управления.

---

## 3. Экран "Список Измерений"

### 3.1. Верхняя панель (Remote Control)
*   **Поведение:** Сворачиваемая (Collapsible).
    *   **Свернуто:** Узкая полоска. Статус Bluetooth, Иконка подключения, Кнопка "Развернуть".
    *   **Развернуто:** Настройки типа замера, Точка отсчета, Кнопка "Включить" (Лазер).
*   **Влияние на список:** При сворачивании список измерений мгновенно занимает освободившееся место.

### 3.2. Список замеров (RecyclerView)
*   **Вид:** Строки в стиле MM (Белый фон, Синий текст значения, Иконка типа слева).
*   **Удаление:** **Отсутствует** иконка корзины в каждой строке.
*   **Навигация:**
    *   Плавающие кнопки (FAB) справа внизу: **⬆ (В начало)**, **⬇ (В конец)**.
    *   Появляются, когда список прокручен от края.

### 3.3. Авто-прокрутка (Smart Scroll)
*   **Новый замер:**
    *   Если пользователь находится **внизу списка** (видит последний элемент) → Список автоматически прокручивается вниз к новому замеру.
    *   Если пользователь смотрит **историю (середина/начало)** → Список **НЕ** прокручивается (чтобы не сбивать с толку).

### 3.4. Режим выделения (Selection Mode)
*   **Вход:** Долгое нажатие на любую строку.
*   **Интерфейс:**
    *   Верхняя панель (ActionMode): Кнопки **[Интервал] [Все] [Удалить] [Отмена]**.
    *   Слева в строках появляются чекбоксы.
    *   Иконки корзин исчезают (если были).
*   **Действия:**
    *   **Интервал:** Тап на первый элемент → Тап на последний элемент (выделяет диапазон). Либо диалог "С ... по ...".
    *   **Все:** Выделяет все замеры.
    *   **Удалить:** Вызывает диалог подтверждения "Удалить N измерений?".
    *   **Переместить:** (Drag & Drop) В этом режиме можно менять порядок строк перетаскиванием.

## 4. Ввод и Редактирование (Подписи/Заметки)

### 4.1. Логика сохранения
*   **Без подтверждения:** Ввод сохраняется автоматически.
*   **Триггеры:** Потеря фокуса (переход на другую строку, сворачивание клавиатуры) ИЛИ пауза в вводе.
*   **Безопасность:** Данные не теряются при переключении приложений (сохраняются в ViewModel/БД сразу).

### 4.2. Позиция курсора
*   **Стандартная подпись** (например, "Прямой замер", "Угол"): Курсор в **начале** строки (для быстрого дописывания). Текст не выделен.
*   **Редактируемая подпись** (уже менялась пользователем): Курсор в **конце** строки.

### 4.3. История версий (Бекап подписей)
*   В БД хранятся 3 версии подписи: `caption_1` (текущая), `caption_2` (пред.), `caption_3` (старая).
*   **Механизм сдвига:** При сохранении новой версии: `3 <- 2 <- 1 <- Новая`.
*   **Восстановление:** Долгое нажатие на поле ввода подписи → Показывает предыдущую версию (`caption_2`) для отката.

### 4.4. Авто-капитализация (Smart Mode)
*   **Режим:** По умолчанию **ВКЛЮЧЕН**.
*   **Логика:**
    *   Приложение анализирует вводимое слово.
    *   Если слово похоже на **технический код/сокращение** (содержит цифры, дефисы, спецсимволы или состоит из 1-3 букв, например: `д-11`, `лп`, `кл7`) → **Принудительно делает** первую букву заглавной (превращает `д-11` в `Д-11`, `лп` в `Лп`).
    *   Если слово обычное → **Не применяет** принудительную капитализацию (оставляет регистр как ввел пользователь).
*   **Цель:** Автоматически исправлять регистр в технических обозначениях, не мешая вводу обычного текста.
