# Логика работы с инклинометром

**Дата:** 4 апреля 2026 г., 02:15 MSK
**Версия:** 1

---

## Принцип

**Инклинометр** — датчик угла наклона в рулетке. Показание приходит в каждом BLE пакете (`angle_deg`).

По этому показанию:
1. **Определяем тип** измерения (прямой/косвенный)
2. **Маркируем** замер в списке
3. **Рассчитываем** высоту/ширину
4. **Предлагаем** перейти в косвенный режим

---

## Определение типа по углу

| Угол (`angle_deg`) | Тип замера | Маркер | Описание |
|--------------------|-----------|--------|----------|
| `-5° ... +5°` | **Прямой замер** | 📏 | Рулетка горизонтально или вертикально |
| `> +5°` | **Косвенная высота** | 📐↑ | Рулетка направлена вверх |
| `< -5°` | **Косвенная ширина** | 📐→ | Рулетка направлена в сторону |

**Порог настраиваемый** (по умолчанию ±5°).

---

## Расчёт размеров

### Прямой замер (|угол| ≤ 5°)

```
result_mm = измеренное расстояние
```

### Косвенная высота (угол > +5°)

```
a = comp1_mm        // измеренное расстояние до точки
b = angle_deg       // угол наклона

высота = a × sin(b)
```

**Пример:**
```
a = 1850.4 мм
b = 51.0°
высота = 1850.4 × sin(51°) = 1850.4 × 0.777 = 1438.3 мм
```

### Косвенная ширина (угол < -5°)

```
a = comp1_mm        // измеренное расстояние до точки
b = |angle_deg|     // угол наклона (модуль)

ширина = a × cos(b)
```

**Пример:**
```
a = 1850.4 мм
b = 51.0°
ширина = 1850.4 × cos(51°) = 1850.4 × 0.629 = 1164.3 мм
```

### Двойная косвенная (2 измерения)

```
a1 = comp1_mm первого замера
a2 = comp2_mm второго замера
b1 = angle_deg первого замера
b2 = angle_deg второго замера

высота = a1 × sin(b1) + a2 × sin(b2)
ширина = a1 × cos(b1) + a2 × cos(b2)
```

## Автоматический переход в косвенный режим

### Логика и Настройки

Пользователь может выбрать один из 4 режимов реакции на угол наклона:

| Режим | Описание | Поведение при угле > X° | Поведение при угле < X° |
|-------|----------|-------------------------|-------------------------|
| **1. Авто** | Полная автоматика | Мгновенно переключает на "Косвенный" | Переключает на "Прямой" |
| **2. Спросить** | Запрос подтверждения | Показывает диалог: [Да, высота] [Да, ширина] [Нет] | Оставляет как есть |
| **3. Голос** | Голосовое подтверждение | Говорит: "Угол 30°. Переходим в косвенный?" → Ждет "Да/Нет" | Оставляет как есть |
| **4. Ручной** | Без автоматики | Ничего не делает (пользователь сам выбирает тип) | Ничего не делает |

> **Порог X°** настраивается (по умолчанию **5°**).

### Влияние на Экспорт (XLSX)

Тип замера, определенный этой логикой, записывается в `measurement_type` и попадает в **основной столбец "Вид"** при экспорте.

**Пример 1: Авто-переход**
1. Замер: 1850 мм, Угол: 51°.
2. Режим: **Авто**.
3. Приложение меняет тип на "Косвенная высота".
4. **Экспорт:** Вид = `Косвенная высота`.

**Пример 2: Отмена пользователем**
1. Замер: 1850 мм, Угол: 51°.
2. Режим: **Спросить**.
3. Пользователь нажимает [Нет] (хочет прямой).
4. Приложение оставляет тип "Прямой замер".
5. **Экспорт:** Вид = `Прямой замер`.

**Пример 3: Обратный переход**
1. Был косвенный замер (Угол 50°).
2. Рулетку опустили (Угол 1°).
3. Режим: **Авто**.
4. Приложение меняет тип обратно на "Прямой замер".
5. **Экспорт:** Вид = `Прямой замер`.

## Хранение в БД

### Таблица `measurements`

| Столбец | Тип | Описание |
|---------|-----|----------|
| `angle_deg` | REAL | **Сырое показание инклинометра** (градусы) |
| `is_auto_detected` | INT | Автоопределение типа (0=ручной, 1=авто) |
| `calculated_value` | REAL | **Рассчитанное значение** (высота/ширина) |
| `calculation_type` | TEXT | Тип расчёта: `direct`, `height`, `width`, `double_height`, `double_width` |

### Примеры строк

#### Прямой замер (автоопределение)
```
angle_deg = 2.3
is_auto_detected = 1
measurement_type = 1 (Direct)
result_mm = 2078.2
calculated_value = NULL
calculation_type = "direct"
```

#### Косвенная высота (автоопределение + расчёт)
```
angle_deg = 51.0
is_auto_detected = 1
measurement_type = 10 (IndHeight)
result_mm = 1444.6       ← результат рулетки (вычисленный)
comp1_mm = 1850.4        ← фактическое измерение a
calculated_value = 1438.3 ← наш расчёт: a × sin(b)
calculation_type = "height"
```

#### Косвенная ширина (ручной выбор)
```
angle_deg = -30.0
is_auto_detected = 0     ← пользователь выбрал вручную
measurement_type = 11 (IndLength)
result_mm = 1165.0       ← результат рулетки
comp1_mm = 1351.9        ← фактическое измерение a
calculated_value = 1169.9 ← наш расчёт: a × cos(b)
calculation_type = "width"
```

## Настройки порогов (храним в БД)

### Таблица: `settings`

| Столбец | Тип | Описание | По умолчанию |
|---------|-----|----------|-------------|
| `key` | TEXT PK | Ключ настройки | — |
| `value` | TEXT | Значение | — |
| `updated_at` | LONG | Время изменения | — |

**Ключи:**
- `inclino_threshold` — порог прямого замера (°), по умолчанию `5`
- `inclino_auto_switch` — автопереключение (0/1), по умолчанию `1`
- `inclino_default_mode` — режим по умолчанию (`height`, `width`, `ask`), по умолчанию `ask`

## Сценарии использования

### Сценарий 1: Прямой замер

1. Пользователь наводит рулетку на стену (угол ~0°)
2. Нажимает кнопку замера
3. Приложение получает: `result_mm=2078.2, angle_deg=2.3`
4. **Автоопределение:** |2.3| ≤ 5° → Прямой замер
5. Сохраняет в БД: `measurement_type=1, calculated_value=NULL`
6. Показывает: `📏 2078.2 мм`

### Сценарий 2: Косвенная высота (авто)

1. Пользователь наводит рулетку вверх (угол > 5°)
2. Нажимает кнопку замера
3. Приложение получает: `comp1_mm=1850.4, angle_deg=51.0`
4. **Автоопределение:** 51.0 > 5° → Показать оповещение
5. Пользователь: "Да, высота"
6. Приложение:
   - Устанавливает `measurement_type=10`
   - Рассчитывает `calculated_value = 1850.4 × sin(51°) = 1438.3`
   - Сохраняет в БД
7. Показывает: `📐↑ 1438.3 мм`

### Сценарий 3: Косвенная ширина (ручной)

1. Пользователь наводит рулетку в сторону
2. Нажимает кнопку замера
3. Приложение получает: `comp1_mm=1351.9, angle_deg=-30.0`
4. **Автоопределение:** -30.0 < -5° → Показать оповещение
5. Пользователь: "Да, ширина"
6. Приложение:
   - Устанавливает `measurement_type=11`
   - Рассчитывает `calculated_value = 1351.9 × cos(30°) = 1169.9`
   - Сохраняет в БД
7. Показывает: `📐→ 1169.9 мм`
