# Qwentest Project — Конфигурация для Qwen Code

## Стек технологий
- **Python**: предпочтительные фреймворки и библиотеки
- **C#**: .NET 6+/8+, ASP.NET Core, консольные приложения

## Правила работы с кодом

### Python
- Использовать type hints (PEP 484)
- Следовать PEP 8 для форматирования
- Предпочитать f-строки для форматирования строк
- Использовать `async/await` для асинхронных операций
- Менеджеры пакетов: `pip`, `poetry` или `uv`

### C#
- Использовать C# 10+ синтаксис (global using, file-scoped namespaces)
- Следовать .NET coding conventions
- Использовать `async/await` для асинхронных операций
- Предпочитать dependency injection
- Использовать pattern matching где уместно

### Python
- Линтер: `ruff` или `flake8`
- Форматтер: `black` или `ruff format`

### C#
- SDK: .NET 8+
- Форматтер: `dotnet format`
- Тесты: `xUnit` или `NUnit`
- Анализ: Roslyn analyzers

## Правила работы

### Конспект сессии
- КОНТЕКСТ ОГРАНИЧЕН (~128K токенов) — терминал НЕ хранит всю сессию от начала до конца
- В ТЕЧЕНИЕ сессии: после каждого важного открытия/решения/действия записывать краткую заметку в `docs/conversations/{номер}_draft.md`
- Формат заметки: `- Что делалось → Результат/Находка`
- draft.md **НЕ УДАЛЯТЬ** — сохранять для каждой сессии как историю работы
- В КОНЦЕ сессии: на основе draft создать FINAL/SUMMARY, обновить CONVERSATIONS.md, DECISIONS.md, .qwen/QWEN.md

### Бэкапы файлов
- Бэкап **только** для файлов, которые будут изменены между коммитами
- Одна копия `.bak` (перезаписывается)
- Перед git commit — `.bak` удаляется через корзину
- `*.bak` и `*.bak.note` в `.gitignore`

## Архивирование диалогов

В конце каждой сессии:
1. **docs/CONVERSATIONS.md** — хронология диалогов с кратким содержанием
2. **docs/conversations/###_YYYY-MM-DD_topic_FULL.md** — для вставки полной копии
3. **docs/conversations/###_YYYY-MM-DD_topic_SUMMARY.md** — резюме с решениями
4. **docs/DECISIONS.md** — новые архитектурные решения
5. **.qwen/QWEN.md** — важные выводы из диалога (этот файл)
6. **docs/conversations/{номер}_draft.md** — конспект сессии (НЕ УДАЛЯТЬ)
